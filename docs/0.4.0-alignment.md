- This file is the authoritative alignment/status matrix for the current branch.
- `docs/SPEC.md`, `docs/cli-contract.md`, `docs/profiles-and-aliases.md`, and most quickstart/reference pages still describe the current 0.3.x/legacy implementation contract unless they explicitly point to the 0.4.0 redesign.
- `docs/UX_REVIEW.md` is a review/rationale artifact, not the final command contract.
- `README.md` remains the public current-usage entry point and may mention `mund` only as branch/current-artifact behavior, not as a released 0.4.0 claim.
- `CHANGELOG.md` is historical release notes; it does not yet contain a 0.4.0 release entry.
- `REVIEW.md` is a status tracker and must not claim release readiness without current verification.

## Release-readiness rule

A 0.4.0 release claim requires all three conditions:

1. `docs/MUND_0_4_UX_SPEC.md` acceptance criteria are implemented or explicitly narrowed with rationale.
2. This matrix has no open row marked `P0 release gate`, `P1 release gate`, or `P1 release decision` unless the row is explicitly narrowed with rationale before release.
3. Fresh verification, including `make test`, code review, and QA evidence, is recorded in `REVIEW.md` or a release PR.
